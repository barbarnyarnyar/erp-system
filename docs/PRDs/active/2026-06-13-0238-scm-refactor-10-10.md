## 1. Objective & Architectural Context

The current **Supply Chain Management (`scm-service`)** utilizes GORM entity layouts and contains partial CDD schema mappings, but it is currently implemented as an **in-memory mock** service. To achieve an industry-grade **10 / 10** rating, we must design the blueprint to transition SCM to a fully GORM-persistent, distributed-resilient architecture that satisfies the five compliance vectors:

1. **Domain Isolation**: Strictly enforce modules boundary control with domain events (e.g. `scm.inventory.reservation.completed`) and zero direct table sharing.
2. **Distributed Consistency & Resilience**: Implement GORM-driven Transactional Outbox pattern persistence with message-retry logging and Event Inbox deduplication.
3. **Clean Architecture & Layering**: Decouple domain services from GORM DB connections using Go's standard `context.Context` and interface boundaries.
4. **Concurrency & Scale Security**: Apply Optimistic Concurrency Control (OCC) with a conditional update version shield on `StockBalance` / `InventoryItem` state changes.
5. **Multi-Tenant Data Safety**: Establish multi-tenant composite primary/unique keys and indexes on `legal_entity_id` at the database constraints level.

### A. Database Tier & Multi-Tenant Isolation
Every table representing operational logistics must have a composite unique key index at the GORM level ensuring tenant data safety:

$$\text{Index}_{\text{composite}} = (\text{legal\_entity\_id}, \text{unique\_domain\_code})$$

Specifically, `StockBalance` must map to `(legal_entity_id, warehouse_id, material_id)`.

### B. Concurrency Shield Implementation
To handle high-velocity stock allocations and prevent split-brain inventory over-allocation, the `StockBalance` struct must utilize **Optimistic Concurrency Control (OCC)**. 
- Struct field: `Version int32`
- Database update queries must conditionally check:

```sql
UPDATE scm_stock_balances 
SET quantity_on_hand = new_on_hand, 
    quantity_allocated = new_allocated, 
    quantity_available = new_available, 
    version = version + 1 
WHERE id = target_id AND version = expected_version;
```

If the rows affected is `0`, the repository layer must abort the transaction and trigger an isolated application retry loop.

### C. Pure Service Decoupling
Domain interfaces (e.g., `InventoryService`) must compile purely using Go's standard library `context.Context` and domain structs:
```go
type InventoryService interface {
	ReserveInventoryStock(ctx context.Context, legalEntityID uuid.UUID, salesOrderID uuid.UUID, itemsToReserve []models.ReservationItemInput) error
	ExecuteStockTransfer(ctx context.Context, transferID uuid.UUID) (*models.StockTransfer, error)
}
```
